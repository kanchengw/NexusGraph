# GraphRAG Production Demo — Architecture & Plan

知识库: RAGBench/techqa | LLM: qwen3.6-flash (DashScope) | Judge: qwen-plus
图数据库: Neo4j 5 | 关系数据库: PostgreSQL 16 | 检索: 3 路召回 (Vector + BM25 + Graph)

## 二、查询链路详细流程图

`mermaid
sequenceDiagram
    participant U as User
    participant API as FastAPI
    participant V as Vector Search
    participant BM as BM25 Search
    participant KG as Graph Expand
    participant PG as PostgreSQL
    participant LF as Langfuse

    U->>API: POST /api/v1/graphrag/query<br/>{question, top_k}
    API->>API: embed_query() -> 1024d vector
    
    par Path 1: Vector Search
        API->>V: db.index.vector.queryNodes()
        V-->>API: top_k chunks with scores
    and Path 2: BM25 Keyword
        API->>BM: db.index.fulltext.queryNodes()
        BM-->>API: top_k chunks with scores
    and Path 3: Graph Expand
        API->>KG: FROM_CHUNK + RELATES_TO traverse
        KG-->>API: related chunks via entities
    end

    API->>API: Merge 3 paths, deduplicate
    API->>PG: INSERT retrieval_metric
    API->>LF: trace + 3 path spans
    API-->>U: {vector, bm25, graph}_context + metrics
`
